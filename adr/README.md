# 🏛️ Architectural Decision Records (adr/)

Este directorio contiene el registro histórico de las decisiones de diseño y teoría aprobadas para la Teoría del Conocimiento Finito (FKT).

## 🧭 Principio Core de Decisiones
*   **Investigación:** Hace preguntas y experimenta.
*   **ADRs:** Registran y justifican decisiones teóricas/de diseño.
*   **Especificaciones:** Formalizan esas decisiones de manera independiente al código.
*   **Implementación:** Ejecuta las especificaciones en software.

---

## 📄 Estructura de un ADR

Cada documento de decisión bajo este directorio debe seguir el formato estándar:

1.  **Título:** `ADR-[Número]: [Nombre de la decisión]` (ej. `ADR-001-bytes-as-primitives.md`).
2.  **Status:** Proposed, Accepted, Rejected, Superceded.
3.  **Context:** ¿Cuál es el problema y qué alternativas existen?
4.  **Decision:** ¿Qué opción se elige y por qué? (referenciando la evidencia científica).
5.  **Consequences:** ¿Qué cambia a partir de ahora?
